# AGENT.md

## 项目定位

本项目是一个用于量化投资学习与研究的多智能体交易分析框架。它通过不同角色的 LLM Agent 模拟投研团队工作流，包括技术分析、基本面分析、新闻分析、情绪分析、交易计划、风险辩论与组合经理决策。

请始终把本项目理解为学习、研究和实验工具，而不是实际投资建议系统。任何分析结果、交易信号、回测结果或报告内容都不应被直接视为买卖建议。

## 重要原则

- 金融安全优先：不要把模型输出包装成确定性收益、保本收益或真实投资建议。
- 学习用途优先：新增功能应帮助理解量化分析、多智能体协作、数据处理或风险管理。
- 可复现实验优先：涉及行情、新闻、财务数据、模型输出和报告生成时，尽量保留日期、数据源、配置和输入参数。
- 小步修改优先：遵循现有目录结构和编码风格，避免无关重构。
- 配置隔离优先：API Key、代理、模型供应商、路径等敏感或环境相关信息应通过 `.env` 或配置项管理，不要硬编码。

## 项目结构速览

- `tradingagents/`：核心多智能体框架。
- `tradingagents/agents/`：分析师、研究员、交易员、风险管理等 Agent 逻辑。
- `tradingagents/graph/`：LangGraph 工作流、状态传播、条件逻辑、断点恢复和信号处理。
- `tradingagents/dataflows/`：行情、新闻、财务数据等数据源适配。
- `tradingagents/llm_clients/`：不同 LLM 服务商的客户端封装、模型目录和校验逻辑。
- `cli/`：交互式命令行入口与展示逻辑。
- `tests/`：单元测试、集成测试和冒烟测试。
- `docs/`：项目架构和运行流程说明。
- `assets/`：README 和文档使用的图片资源。
- `reports/`、`results/`：本地运行产生的分析结果或示例输出。

## 常用命令

安装项目：

```bash
pip install .
```

运行 CLI：

```bash
tradingagents
```

或从源码运行：

```bash
python -m cli.main
```

运行测试：

```bash
pytest
```

运行冒烟测试：

```bash
python scripts/smoke_structured_output.py
```

Docker 运行：

```bash
cp .env.example .env
docker compose run --rm tradingagents
```

## 开发约定

- Python 版本要求见 `pyproject.toml`，当前为 `>=3.10`。
- 依赖优先维护在 `pyproject.toml` 中。
- 新增测试放在 `tests/`，文件名遵循 `test_*.py`。
- 修改 Agent 输出结构、状态字段或决策流程时，应同步检查 `tradingagents/agents/schemas.py`、`tradingagents/graph/` 和相关测试。
- 修改数据源逻辑时，应关注网络失败、限流、空数据、交易日缺失和 ticker 格式兼容。
- 修改 CLI 时，应兼顾中文输出、终端编码和 Windows/macOS/Linux 的兼容性。
- 不要提交 `.env`、本地密钥、缓存文件、虚拟环境或大体积运行产物。

## 量化投资学习注意事项

- 所有策略、信号和报告都需要区分“历史解释”“模型推断”和“未来预测”。
- 回测结果必须警惕过拟合、幸存者偏差、前视偏差、手续费遗漏、滑点遗漏和数据质量问题。
- 单次 Agent 决策不具备统计显著性，建议结合多日期、多标的、多市场环境观察。
- LLM 输出可能出现幻觉，涉及财报、新闻、宏观事件和监管信息时应尽量保留来源并进行人工核验。
- 风险管理模块不应被弱化；新增策略逻辑时，要同步考虑仓位、止损、波动率、流动性和极端事件。

## AI 协作指南

当 AI 助手参与本项目开发时：

- 先阅读 `README.md`、`docs/tradingagents_architecture.md` 和相关模块代码，再修改实现。
- 优先保持现有多智能体角色划分，不要随意合并职责清晰的 Agent。
- 若要新增 Agent，应说明它在分析链路中的输入、输出、依赖数据源和风险边界。
- 若要新增模型服务商，应放在 `tradingagents/llm_clients/`，并补充模型校验和必要测试。
- 若要新增数据源，应放在 `tradingagents/dataflows/`，并处理异常、缺失值和速率限制。
- 对金融结论类文案保持克制，避免使用“必涨”“稳赚”“确定买入”等表达。
- 修改后优先运行与变更范围相关的测试；范围较大时运行完整 `pytest`。

## 输出与报告

CLI 和 Python API 运行后通常会生成本地报告、状态日志或决策记录。报告可用于学习和复盘，但需要结合原始数据、配置、模型版本和运行日期理解。

报告生成脚本包括：

- `generate_report.py`
- `visualize_report.py`

如果调整报告结构或字段，请同步检查生成脚本、CLI 展示和测试。
